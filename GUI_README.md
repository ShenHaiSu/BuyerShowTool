# BuyerShowTool GUI 使用说明

## 概述

BuyerShowTool GUI 是一个基于 Tkinter 的图形用户界面程序，用于生成买家秀图片和好评文案。

## 功能特性

- **图形化界面**：直观的操作界面，替代命令行操作
- **配置管理**：轻松配置 API 密钥和生成参数
- **任务管理**：批量处理多个商品目录
- **实时反馈**：实时显示任务进度和日志信息
- **Windows 优化**：适配 Windows 系统特性

## 使用方法

### 1. 启动程序

**方法一：使用启动脚本**
双击运行 `run_gui.bat` 文件。

**方法二：直接运行可执行文件**
双击运行 `dist/BuyerShowTool.exe` 文件。

### 2. 配置 API 密钥

1. 打开"配置页面"标签页
2. 输入 DeepSeek API Key
3. 输入通义万相 API Key
4. 点击"保存配置"按钮

### 3. 选择目标目录

1. 点击工具栏中的"选择目录"按钮
2. 选择包含商品配置文件（goods.yaml）的根目录
3. 程序会自动扫描并显示所有符合条件的子目录

### 4. 开始生成

1. 确保已配置 API 密钥
2. 点击"开始生成"按钮
3. 程序会在后台处理所有任务
4. 可以在"任务页面"查看进度和日志

### 5. 停止任务

点击"停止"按钮可以中断当前任务。

## 文件结构

```
BuyerShowTool/
├── dist/
│   └── BuyerShowTool.exe    # 可执行文件
├── config.yaml               # 配置文件（自动生成）
├── config.example.yaml       # 配置示例文件
└── README.md                 # 项目说明
```

## 注意事项

1. **配置文件**：首次运行时会自动生成 `config.yaml` 文件
2. **API 密钥**：需要配置有效的 DeepSeek 和通义万相 API 密钥
3. **商品配置**：目标目录需要包含 `goods.yaml` 文件
4. **网络连接**：需要稳定的网络连接以调用 API

## 故障排除

### 配置文件不存在

如果提示配置文件不存在，请：
1. 检查 `config.yaml` 文件是否存在
2. 如果不存在，复制 `config.example.yaml` 为 `config.yaml`
3. 修改 `config.yaml` 中的 API 密钥配置

### 商品配置文件不存在

如果提示商品配置文件不存在，请：
1. 确保目标目录包含 `goods.yaml` 文件
2. 如果不存在，程序会自动生成模板文件
3. 编辑 `goods.yaml` 文件，配置商品信息

### API 调用失败

如果 API 调用失败，请：
1. 检查 API 密钥是否正确
2. 检查网络连接是否正常
3. 查看日志输出获取详细错误信息
