# BuyerShowTool GUI 开发总结

## 项目概述

本项目成功将 BuyerShowTool 的命令行界面（CLI）转换为图形用户界面（GUI），基于 Tkinter 框架开发，运行于 Windows 操作系统。

### 阶段 2: 核心功能集成 ✅

## 功能特性

### 1. 用户友好界面

- 经典的 "标题栏 + 菜单栏 + 工具栏 + 主工作区 + 状态栏" 布局
- 使用 ttk.Notebook 实现标签页切换
- 实时日志输出显示
- 任务进度实时更新

### 2. 配置管理

- API 密钥配置（DeepSeek、通义万相）
- 生成参数配置（好评文案数量、买家秀图片数量）
- 配置文件自动保存和加载

### 3. 任务管理

- 目录扫描和任务列表展示
- 后台线程处理任务，避免阻塞 UI
- 实时进度更新和状态显示
- 支持任务停止功能

### 4. Windows 优化

- 适配 Windows 文件路径处理
- 使用 Windows 风格的文件对话框
- 支持高 DPI 显示
- 使用现代化的 ttk 主题

## 技术实现

### 线程安全通信

- 使用 `queue.Queue` 在后台线程和主线程之间传递日志和进度信息
- 主线程定时从队列中读取信息并更新 UI

### 日志重定向

- 自定义 `sys.stdout` 和 `sys.stderr` 类
- 将输出写入队列，由主线程更新 UI

### 配置管理

- 使用 PyYAML 解析和保存配置文件
- 支持配置验证和默认值

## 使用方法

### 1. 启动程序

双击运行 `dist/BuyerShowTool.exe` 或使用 `run_gui.bat` 脚本。

### 2. 配置 API 密钥

1. 打开"配置页面"标签页
2. 输入 DeepSeek API Key
3. 输入通义万相 API Key
4. 点击"保存配置"按钮

### 3. 选择目标目录

1. 点击工具栏中的"选择目录"按钮
2. 选择包含商品配置文件（goods.yaml）的根目录

### 4. 开始生成

1. 确保已配置 API 密钥
2. 点击"开始生成"按钮
3. 在"任务页面"查看进度和日志

## 注意事项

1. **配置文件**：首次运行时会自动生成 `config.yaml` 文件
2. **API 密钥**：需要配置有效的 DeepSeek 和通义万相 API 密钥
3. **商品配置**：目标目录需要包含 `goods.yaml` 文件
4. **网络连接**：需要稳定的网络连接以调用 API
