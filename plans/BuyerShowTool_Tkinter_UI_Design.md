# BuyerShowTool Tkinter UI 设计方案

## 一、项目概述

本文档旨在为 BuyerShowTool 项目设计一个基于 Tkinter 的图形用户界面（GUI），运行于 Windows 操作系统。该 GUI 将替代现有的命令行界面（CLI），提供更直观、易用的操作体验。

### 1.1 现有 CLI 逻辑分析

通过分析 `src/index.py` 及相关模块，核心业务流程如下：

1.  **初始化与配置检查**：
    *   检查 `config.yaml` 是否存在，若不存在则生成 `config.example.yaml` 并提示用户。
    *   检查目标目录下是否存在 `goods.yaml`，若不存在则生成模板并提示。
2.  **目录遍历**：
    *   使用 `find_goods_yaml_paths` 递归查找包含 `goods.yaml` 的子目录。
3.  **任务处理**：
    *   对每个目标目录，加载配置和商品信息。
    *   调用 `ReviewGenerator` 生成好评文案（DeepSeek API）。
    *   调用 `ImageGenerator` 生成买家秀图片（通义万相 API）。
    *   保存结果（`review.txt`, `buyer_show_N.jpg`）。
4.  **错误处理**：
    *   使用 `ErrorHandler` 记录错误并生成日志文件。

### 1.2 设计目标

*   **用户友好**：提供图形化界面，简化配置和操作流程。
*   **功能完整**：保留 CLI 的所有核心功能。
*   **实时反馈**：在 GUI 中实时显示任务进度和状态。
*   **Windows 优化**：适配 Windows 系统特性（如文件路径、窗口样式）。
*   **可扩展性**：代码结构清晰，便于后续功能扩展。

*   **编程语言**：Python 3.8+

## 三、UI 界面设计

### 3.1 窗口布局

主窗口采用经典的 "标题栏 + 菜单栏 + 工具栏 + 主工作区 + 状态栏" 布局。

```
+-------------------------------------------------------+
| BuyerShowTool - 买家秀生成工具          [最小化] [关闭] |
+-------------------------------------------------------+
| [文件] [帮助]                                         |
+-------------------------------------------------------+
| [选择目录] [开始生成] [停止]                           |
+-------------------------------------------------------+
|                                                       |
|  主工作区 (Notebook)                                  |
|  +-------------------+  +-------------------+         |
|  |   配置页面        |  |   任务页面        |         |
|  |                   |  |                   |         |
|  +-------------------+  +-------------------+         |
|                                                       |
+-------------------------------------------------------+
| 就绪 | 进度: 0% | 成功: 0 | 失败: 0                    |
+-------------------------------------------------------+
```

### 3.2 详细界面组件

#### 3.2.1 菜单栏 (Menu Bar)

*   **文件 (File)**:
    *   `打开配置文件`: 使用系统默认编辑器打开 `config.yaml`。
    *   `打开日志目录`: 打开项目根目录。
    *   `退出`: 关闭程序。
*   **帮助 (Help)**:
    *   `关于`: 显示版本信息和项目链接。
    *   `使用指南`: 打开项目文档。

#### 3.2.2 工具栏 (Toolbar)

*   **选择目录按钮**: 弹出文件夹选择对话框，选择目标根目录。
*   **开始生成按钮**: 启动任务处理流程。
*   **停止按钮**: 停止当前任务（设置停止标志）。

#### 3.2.3 主工作区 (Main Area)

使用 `ttk.Notebook` 实现标签页切换。

**标签页 1: 配置页面 (Configuration)**

*   **API 配置区域**:
    *   DeepSeek API Key (输入框，掩码显示)。
    *   通义万相 API Key (输入框，掩码显示)。
    *   `保存配置` 按钮。
*   **生成参数区域**:
    *   好评文案生成数量 (SpinBox, 范围 1-10)。
    *   买家秀图片生成数量 (SpinBox, 范围 1-10)。
*   **目标目录显示**:
    *   显示当前选择的目录路径。

**标签页 2: 任务页面 (Tasks)**

*   **任务列表 (Treeview)**:
    *   列: 序号, 目录路径, 状态 (待处理/处理中/成功/失败), 进度, 耗时。
*   **日志输出区域 (Text Widget)**:
    *   实时显示控制台输出的日志信息（通过重定向 `sys.stdout`）。
*   **预览区域 (可选)**:
    *   显示当前处理目录的商品图片和生成的买家秀图片缩略图。

#### 3.2.4 状态栏 (Status Bar)

*   **左侧**: 当前状态信息（如 "就绪"、"处理中..."）。
*   **中间**: 总体进度条。
*   **右侧**: 统计信息（成功/失败数量）。

## 四、核心模块设计

为了将 CLI 逻辑适配到 GUI，需要对现有代码进行封装和扩展。

### 4.1 主程序入口 (`gui_main.py`)

*   **职责**: 初始化 Tkinter 窗口，启动主事件循环。
*   **代码结构**:
    ```python
    import tkinter as tk
    from tkinter import ttk
    from ui.main_window import MainWindow

    def main():
        root = tk.Tk()
        root.title("BuyerShowTool - 买家秀生成工具")
        root.geometry("1000x700")
        
        # 设置 Windows 图标 (可选)
        # root.iconbitmap("assets/icon.ico")
        
        app = MainWindow(root)
        root.mainloop()

    if __name__ == "__main__":
        main()
    ```

### 4.2 主窗口类 (`ui/main_window.py`)

*   **职责**: 管理整个窗口的布局、组件和事件绑定。
*   **关键属性**:
    *   `task_manager`: 任务管理器实例，负责后台任务执行。
    *   `is_running`: 布尔值，标记任务是否正在运行。
*   **关键方法**:
    *   `create_widgets()`: 创建菜单、工具栏、Notebook 等。
    *   `on_select_directory()`: 处理目录选择逻辑。
    *   `on_start_generation()`: 启动任务生成流程。
    *   `on_stop_generation()`: 停止任务生成流程。
    *   `update_status()`: 更新状态栏信息。

### 4.3 任务管理器 (`ui/task_manager.py`)

*   **职责**: 封装 CLI 的核心逻辑，运行在后台线程中，避免阻塞 UI。
*   **关键方法**:
    *   `run_tasks(target_dir: str)`: 遍历目录并处理任务。
    *   `process_single_target(target_dir: str)`: 调用 `ReviewGenerator` 和 `ImageGenerator`。
    *   `update_progress(callback)`: 进度更新回调函数。
*   **线程安全**:
    *   使用 `queue.Queue` 在后台线程和主线程之间传递日志和进度信息。
    *   主线程定时从队列中读取信息并更新 UI。

### 4.4 配置管理器 (`ui/config_manager.py`)

*   **职责**: 管理 `config.yaml` 的读取和保存。
*   **关键方法**:
    *   `load_config()`: 加载配置到 UI 控件。
    *   `save_config()`: 从 UI 控件保存配置到文件。
    *   `validate_config()`: 验证配置的有效性。

### 4.5 日志重定向 (`ui/logger.py`)

*   **职责**: 将控制台日志重定向到 Tkinter Text 组件。
*   **实现**:
    *   自定义 `sys.stdout` 和 `sys.stderr` 类，将输出写入 `queue.Queue`。
    *   主线程定时读取队列并更新 Text 组件。

## 五、Windows 操作系统适配

### 5.1 文件路径处理

*   **问题**: Windows 使用反斜杠 `\` 作为路径分隔符，而 Python 字符串中 `\` 是转义字符。
*   **解决方案**:
    *   使用 `os.path.join()` 拼接路径。
    *   使用 `pathlib.Path` 处理路径（更现代、更安全）。
    *   在 GUI 中显示路径时，确保使用正确的分隔符。

### 5.2 文件对话框

*   **使用 Tkinter 标准库**:
    *   `tkinter.filedialog.askdirectory()`: 选择目录。
    *   `tkinter.filedialog.askopenfilename()`: 选择文件。
*   **优化**:
    *   设置初始目录为用户文档目录或上次选择的目录。
    *   使用 `tkinter.Tk().withdraw()` 隐藏主窗口，仅显示对话框。

### 5.3 窗口样式

*   **主题**: 使用 `ttk.Style()` 设置现代化的主题（如 'vista' 或 'xpnative'）。
*   **图标**: 加载 Windows 风格的 `.ico` 图标文件。
*   **高 DPI 支持**: 在 Windows 10/11 上，确保 Tkinter 窗口在高分辨率屏幕上显示清晰。
    *   可以通过设置 `tkinter.Tk().tk.call('tk', 'scaling', scale_factor)` 调整缩放比例。

### 5.4 进程管理

*   **后台任务**: 使用 `threading.Thread` 执行耗时任务（API 调用、文件处理）。
*   **进程终止**: 提供 "停止" 按钮，通过设置标志位安全终止后台线程。

## 六、交互流程设计

### 6.1 启动流程

1.  用户双击 `gui_main.py` 启动程序。
2.  程序加载默认配置，显示主窗口。
3.  检查 `config.yaml` 是否存在：
    *   若不存在，弹出提示框，引导用户配置 API Key。
    *   若存在，自动填充配置页面的输入框。

### 6.2 任务执行流程

1.  用户点击 "选择目录"，弹出文件夹选择对话框。
2.  用户选择目标根目录，程序自动扫描包含 `goods.yaml` 的子目录。
3.  扫描结果展示在 "任务页面" 的 Treeview 中。
4.  用户点击 "开始生成"：
    *   按钮变为 "停止"，禁用目录选择。
    *   后台线程开始逐个处理目录。
    *   实时更新进度条和日志输出。
5.  任务完成后，按钮恢复为 "开始生成"，启用目录选择。
6.  用户可以查看生成的文件（在目标目录下）。

### 6.3 错误处理流程

1.  任务执行过程中发生错误（如 API 调用失败）。
2.  错误信息通过队列传递到主线程。
3.  主线程在日志区域显示错误信息，并在 Treeview 中标记该任务为 "失败"。
4.  任务继续执行下一个目录（不中断整个流程）。
5.  任务结束后，弹出提示框显示统计信息（成功/失败数量）。
